Section 1

What is PixIQ?

PixIQ is an AI-powered industrial vision controller developed by DhvaniAI Technologies. It is a complete, pre-integrated hardware and software platform that enables manufacturers to deploy intelligent visual inspection systems on the factory floor — in days rather than months.

Traditional vision system deployments require assembling a custom stack: selecting hardware, writing acquisition software, integrating AI frameworks, connecting to PLCs, building an operator interface, and wiring everything to a cloud backend. This typically takes a specialist team 6–12 months. PixIQ packages all of this into a single, validated system that arrives ready to connect.

The core idea: PixIQ is a DIY-friendly vision platform. A customer with cameras, a production line, and a quality problem can go from unboxing to live inspection in days — without a large engineering team.

Section 2

The Problem It Solves

Deploying a vision inspection system on a production line has historically required:

  • A machine vision engineer to design and integrate the hardware
  • A software team to build acquisition, inference, and communication pipelines
  • An AI team to train, validate, and deploy defect detection models
  • A PLC integrator to connect the vision system to factory automation
  • Months of commissioning, debugging, and validation

The result is a bespoke system that is expensive to build, difficult to replicate across lines, and dependent on the original engineering team for updates and maintenance.

PixIQ collapses this into a single product. The hardware arrives pre-configured. The software stack is pre-integrated. AI models are trained in the cloud and deployed over the air. Factory connections use standard industrial protocols that any PLC engineer already understands. No custom software development is required to get to production.

The shift: From "build a vision system" to "configure a vision platform." The difference is the same as buying a smartphone vs. building a phone from components — the underlying technology is just as powerful, but the time-to-value collapses dramatically.
Section 3

System Architecture

PixIQ uses a dual-system architecture that physically separates real-time AI processing from the operator interface. This is a deliberate design choice: the AI controller runs nothing but inspection. It has no screen, no browser, no user interface competing for its GPU and memory. Every CPU cycle and every byte of shared memory is dedicated to getting the inspection result out in under 100 milliseconds.

🧠 Vision Controller

NVIDIA Jetson Orin / Asus PE1100N

  • Camera image acquisition (CUDA-accelerated)
  • AI inference engine (TensorRT, <40ms)
  • PLC communication — PASS/FAIL signals
  • Hardware trigger handling
  • Results decision logic

🖥️ Operator HMI

Asus AIO A3202 / Industrial Panel PC

  • Live inspection feed with defect overlays
  • Production statistics dashboard
  • Local database and audit trail
  • MES / ERP integration gateway
  • Multi-role access control

The two systems communicate over a local Ethernet connection. The Vision Controller pushes inspection results and images to the HMI asynchronously — the critical path (camera → AI → PLC) never waits for the HMI. If the HMI is rebooting or offline, inspection continues uninterrupted.

Real-Time Performance Design

The inspection critical path — from hardware trigger to PLC response — is engineered for deterministic sub-100ms performance. Every component on this path runs natively on the controller hardware with the highest process priority, direct GPU memory access, and pre-allocated image buffers. Nothing on the critical path runs inside a container or shares resources with non-inspection services.

Supported Cameras

PixIQ supports the most widely used industrial camera manufacturers out of the box, with no custom driver development required:

  • Basler (GigE, USB3)
  • The Imaging Source

Section 4

Two Operating Modes

Every PixIQ device operates in one of two modes. Switching between them is done from the HMI without any engineering intervention.

Mode 1

Inspection Mode

The system runs live production inspection. Every part that passes through the camera field of view is analysed by the AI model and a PASS, FAIL, or VERIFY result is sent to the PLC within 100ms. All images and results are logged to the local database and streamed to the HMI in real time.

Mode 2

Training Mode

The system captures and uploads images to the DhvaniAI cloud platform for annotation and model training. Once a new model is trained on A100 GPU infrastructure in the cloud, it can be deployed back to the PixIQ device with a single click — no local engineering required.

The training loop in practice: Run in Training Mode for a few shifts to collect images of your part. Upload to the cloud. Our team (or yours) annotates good and defective examples. A new model is trained and validated. One click deploys it to the device. Switch back to Inspection Mode.
Section 5

AI Capabilities

PixIQ supports multiple AI model types, each suited to a different class of inspection problem. The right model is selected during the setup phase based on the inspection task.

🔍
Anomaly Detection
Trains on good parts only. Flags anything that deviates from normal. Best for new production lines or where defect examples are scarce. No defect images needed to go live.
🗺️
Defect Segmentation
Pixel-level localisation of defect regions (U-Net architecture). Used when defect location and size matter — such as measuring crack length or marking the exact defect position on the image.
📦
Object Detection
Identifies and locates specific features or objects in the image (YOLO architecture). Used for part presence/absence checks, orientation verification, and assembly inspection.
🔤
OCR / Text Reading
Reads printed text, laser-etched codes, date stamps, and part numbers directly from the production line. Used for traceability and label verification.
🏷️
Classification
Categorises parts or defects into defined classes (e.g., crack vs. lap vs. seam). Used when the type of defect matters for downstream process decisions.
📐
Dimension Inspection
Data-driven dimensional verification using vision — checking gap widths, hole diameters, and part geometry without physical gauges. In active development.
Model selection is not permanent. Because models are managed in the cloud and deployed over the air, a PixIQ device on a connecting rod line today can be reconfigured for a camshaft line tomorrow — without any hardware change and without an engineer on site.
Section 6

Factory Integration

PixIQ is designed to drop into an existing production line without disrupting the surrounding automation. It speaks the same protocols that factory PLCs already use, and it outputs standard digital signals that any rejection mechanism can act on.

PLC Communication

  • Modbus TCP/IP — the most widely supported industrial protocol, compatible with virtually all PLC brands
  • OPC-UA — for modern SCADA and MES integrations requiring richer data exchange
  • EtherNet/IP — for Allen-Bradley and Rockwell Automation environments
  • Digital I/O (24VDC) — direct wiring for simple PASS/FAIL signals and trigger inputs

Triggering

Inspection is triggered by a proximity sensor or PLC signal the moment a part enters the camera field of view. Hardware triggering ensures the camera captures at exactly the right moment — synchronisation accuracy is under 1ms across all connected cameras simultaneously.

Rejection Control

On a FAIL result, PixIQ sends an immediate digital output signal to the rejection mechanism — typically a pneumatic gate or diverter. On a VERIFY result, the part is routed to a manual review station where the operator sees the AI-highlighted image and makes the final call. On PASS, the line continues with no interruption.

MES / ERP Integration

The HMI exposes a REST API that allows L3 manufacturing execution systems and ERP platforms to pull inspection data in real time — part serial numbers, inspection timestamps, results, defect coordinates, and images. This enables factory-wide quality dashboards without any custom middleware.

Section 7

Cloud Management

Every PixIQ device is connected to the DhvaniAI cloud platform. From a single dashboard, engineers and quality managers can oversee an entire fleet of deployed devices across multiple production lines and facilities.

Device Hierarchy

Each device in the cloud platform is organised into three levels:

  • Workflows — the inspection processes configured on the device (e.g., "top-face inspection", "bore inspection")
  • Data — all images, results, and logs captured by the device
  • Models — the AI models assigned to each workflow, with version history

One-Click Model Deployment

When a new or updated AI model is ready, it is pushed to one or many devices simultaneously from the cloud dashboard. The device downloads the model, validates it, and hot-swaps it into the inspection pipeline — without stopping production. The previous model version is retained and can be rolled back instantly if needed.

Over-the-Air Software Updates

The full PixIQ software stack on both the Vision Controller and HMI is managed through Azure IoT Edge. Software updates, configuration changes, and security patches are pushed over the air without requiring an engineer to visit the site.

Fleet-Wide Analytics

The cloud platform aggregates inspection data across all connected devices. Quality managers can compare defect rates across production lines, track model performance over time, identify systemic process issues across facilities, and generate compliance reports — all from a single interface.

Multi-site scaling: Once a model is validated on one production line, replicating it to 10 more lines across 3 facilities is a single cloud operation. No engineer travel required.
